WebHooks Integration

Streamline Your Restaurant’s Data Exchange

Fooodis WebHooks enable seamless integration between your restaurant's external systems and the Fooodis platform. With WebHooks, you can receive real-time notifications for key events like order approvals or status changes, ensuring your system stays synchronized. By simply configuring WebHook URLs, your system will receive automated POST requests containing critical data to enhance your restaurant’s operational efficiency.

WebHooks Integration

Seamless Data Synchronization between Fooodis and Your System
Access WebHooks:
  ○ Click on the Apps menu in the left sidebar and select WebHooks from the submenu.

Enter WebHook URLs:
  ○ Provide the URLs where your system expects to receive notifications.

Webhook called when order is approved by system:
  ○ Receives a POST request when the system approves an order.

Webhook called when order is approved by vendor:
  ○ Receives a POST request when the vendor approves an order.

Webhook called when order status changes:
  ○ Receives a POST request containing the order information and last status.

Save Settings:
  ○ Click the Save button to apply your WebHook URLs.

Best Practices for Using WebHooks:
  ○ Make sure your WebHook endpoint is secure and accessible.
  ○ Handle WebHook requests promptly to ensure smooth synchronization.
  ○ Log all incoming WebHook requests for debugging and auditing purposes

Common Questions about WebHooks Integration

Fooodis WebHooks enable seamless data exchange between your restaurant’s external systems and the Fooodis platform. Here are answers to frequently asked questions to help you get the most out of this feature.

What are WebHooks, and how do they work in Fooodis?
  • WebHooks are automated notifications sent to your system when specific events occur in Fooodis. Each WebHook sends a POST request containing data related to the event.
What WebHooks are available in Fooodis?
  • Order Approved by System: Triggered when an order is approved by the Fooodis system.
  • Order Approved by Vendor: Triggered when an order is approved by the restaurant/vendor.
  • Order Status Changes: Triggered when the status of an order changes.
How do I secure my WebHook endpoints?
  • Implement authentication or token validation to verify the legitimacy of incoming requests. Use SSL/TLS encryption to protect data in transit.
How can I test my WebHook setup?
  • Use tools like Postman or create a mock server to simulate and test incoming WebHook requests.
What should I do if my WebHook endpoint is temporarily unavailable?
  • Make sure your WebHook endpoint can handle retries or queue failed requests. Log incoming requests to troubleshoot issues.
Can I receive WebHook notifications for multiple systems?
  • Yes, you can set up multiple WebHook URLs to receive notifications for different systems or purposes.
How quickly will I receive WebHook notifications?
  • WebHook notifications are sent in real time, immediately after the triggering event occurs in Fooodis.
