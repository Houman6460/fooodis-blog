​WebHooks-Integrering

Effektivisera ditt restaurangens datautbyte

​Fooodis WebHooks möjliggör en smidig integration mellan dina externa system och Fooodis-plattformen. Med WebHooks kan du ta emot realtidsnotifikationer för viktiga händelser, som ordergodkännanden eller statusändringar, vilket säkerställer att ditt system förblir synkroniserat. Genom att konfigurera WebHook-URL:er kan ditt system automatiskt ta emot POST-förfrågningar med viktig data, vilket förbättrar din restaurangs operativa effektivitet.

WebHooks Integration

Sömlös data synkronisering mellan Fooodis och ditt system
Access WebHooks:
​  ○Klicka på Apps-menyn i den vänstra sidofältet och välj WebHooks från undermenyn.  

Ange WebHook-URL:​ 
 ○Specify the web addresses where your system is expected to receive notifications.

Webhook som anropas när beställningen godkänns av systemet:​
  ○Tar emot en POST-begäran när systemet godkänner en beställning.

Webhook called when order is approved by vendor:

​  ○ Tar emot en POST-begäran när säljaren godkänner en beställning.

Webhook som anropas när beställningsstatus ändras:

​  ○ Tar emot en POST-begäran som innehåller orderinformation och senaste status.

Spara inställningar:
​  ○ Klicka på Spara-knappen för att tillämpa dina WebHook-URL:er.

Bästa praxis för användning av WebHooks:

  ○ Se till att din WebHook-ändpunkt är säker och åtkomlig. 
  ○ Hantera WebHook-begäranden snabbt för att säkerställa smidig synkronisering. 
  ○ Logga alla inkommande WebHook-begäranden för felsökning och granskningändamål.

​Vanliga Frågor om WebHooks-Integrering

​Fooodis WebHooks möjliggör smidigt datautbyte mellan dina externa system och Fooodis-plattformen. Här är svar på de vanligaste frågorna för att hjälpa dig att maximera nyttan av denna funktion.

​Vad är WebHooks och hur fungerar de i Fooodis?
  • ​WebHooks är automatiska notifikationer som skickas till ditt system när specifika händelser inträffar i Fooodis. Varje WebHook skickar en POST-förfrågan med data relaterad till händelsen.
​Vilka WebHooks finns tillgängliga i Fooodis?
  • ​Order godkänd av systemet: Utlöses när en order godkänns av Fooodis-systemet.
  • Order godkänd av leverantören: Utlöses när en order godkänns av restaurangen/leverantören.
  • Orderstatus ändras: Utlöses när statusen för en order ändras.
​Hur kan jag säkra mina WebHook-slutpunkter?
  • ​Implementera autentisering eller tokenvalidering för att verifiera äktheten hos inkommande förfrågningar. Använd SSL/TLS-kryptering för att skydda data under överföring.
​Hur kan jag testa min WebHook-konfiguration?
  • ​Använd verktyg som Postman eller skapa en mockserver för att simulera och testa inkommande WebHook-förfrågningar.
​Vad ska jag göra om min WebHook-slutpunkt tillfälligt är otillgänglig?
  • ​Säkerställ att din WebHook-slutpunkt kan hantera omförsök eller köa misslyckade förfrågningar. Logga alla inkommande förfrågningar för felsökning.
​Kan jag ta emot WebHook-notifikationer för flera system?
  • ​Ja, du kan konfigurera flera WebHook-URL:er för att ta emot notifikationer för olika system eller ändamål.
​Hur snabbt får jag WebHook-notifikationer?
  • ​WebHook-notifikationer skickas i realtid, direkt efter att den utlösande händelsen inträffar i Fooodis.
